# ADR 0002 — Prisma 7 driver adapter + prisma.config.ts

- **Status:** accepted
- **Date:** 2026-05-27
- **Milestone:** M0

## Context

The project installed Prisma 7 (latest). Prisma 7's Rust-free client changes two
things the draft schema in [`09-data-schema.md`](../09-data-schema.md) did not
anticipate:

1. `url = env("DATABASE_URL")` is **no longer allowed** in the `datasource`
   block of `schema.prisma`.
2. The runtime client needs a **driver adapter**; CLI/migration commands read the
   connection URL from a `prisma.config.ts` file.

## Decision

- Generate with the new `prisma-client` generator into `src/generated/prisma`
  (gitignored; regenerated by a `postinstall` hook).
- Provide the runtime connection via `@prisma/adapter-pg` (`PrismaPg`) in
  `src/server/db.ts`.
- Configure migrations/CLI via `prisma.config.ts`, which loads `.env`
  (`dotenv/config`) and passes `datasource.url` from `DATABASE_URL`.

## Consequences

- A Postgres driver (`pg`) is a runtime dependency.
- `DATABASE_URL` must be present in the environment for `prisma generate`
  (the config resolves `env("DATABASE_URL")` at load time). CI sets it job-wide;
  local dev uses `.env`.
- The generated client is imported from `@/generated/prisma/client` rather than
  `@prisma/client`.
- If this proves cumbersome, pinning to Prisma 6 (legacy `prisma-client-js`,
  `url` in schema) is the fallback. Not expected to be necessary.
